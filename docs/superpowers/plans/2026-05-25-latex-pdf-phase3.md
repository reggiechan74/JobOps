# latex-pdf Skill — Phase 3 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development.

**Goal:** Complete the `latex-pdf` skill so it fully replaces `markdown-to-pdf` and `convert-to-pdf`. Adds the generic `document` doctype (clean general-purpose preamble for rubrics / briefings / career-analysis / crisis-management outputs), accepts a single file / glob / directory as `$1` (matching the old `/pdf` shape), and replaces the two old converters with thin deprecation stubs that delegate.

**Architecture:** Same Phase-1/2 pipeline extended in three orthogonal ways:
1. Third preamble template (`preamble.document.tex.template`) — minimal, clean, no resume-specific macros or cover-letter table styling.
2. `$1` is expanded at the very top of the skill: directory → `*.md` non-recursive list; glob → bash glob expansion; file → list of one. Pipeline runs per file.
3. Doctype autodetect gets a `document` fall-through default. Previous defaults (resume) only apply when explicit signals exist (front-matter `resume_*`, parent dir `resume/`, filename contains `step{1,2,3}` or `resume`).

**Tech Stack:** Same as Phase 1/2 — no new system dependencies.

**Source spec:** `SourceMaterial/JobOps_MarkdownToLatexPdf_Skill_Kickstarter.md` — section "REPLACEMENT SCOPE" calls out three doctypes and the `/pdf` takeover. Section "MIGRATION / REPLACEMENT PLAN" specifies thin deprecation stubs that delegate (NOT silent removal).

**Out of scope for Phase 3:**
- Optional Lua filter for hand-crafted resume `\role`/`\subrole` macro fidelity (kickstarter calls it optional).
- `signature_image` consumption for coverletter (still reserved per Phase 2 comment).
- Removal of the old skills' source files (Phase 4 — one release later).
- `plugin.json` version bump and marketplace.json (release time, separate step via `/version-bump`).

if [ -d "$input" ]; then
  # Directory: list *.md non-recursively
  shopt -s nullglob
  sources=( "$input"/*.md )

elif [[ "$input" == *[\*\?]* ]]; then
